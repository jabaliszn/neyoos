## K.10 — Parent Uploads & Approval Workflows  ✅ REPAIRED & VERIFIED

**Problems found (over-claimed `[~]`):**
- `student-approval.service.ts` called `tenantDb()` WITHOUT a `withTenant` wrapper → would throw "No tenant in scope" at runtime.
- No ownership / row-scope checks: comment literally said *"In a real app we'd explicitly check if the PARENT owns this student."*
- Route used the invalid plural permission `students.view` (the real one is `student.view`).
- No proven parent-portal upload UI; no proven department/teacher photo-edit restriction.

**Fix (full-stack):**
- Service rewritten: every function wrapped in `withTenant(user.tenantId, ...)`; submit/review/history now enforce ownership via `canViewStudent` (parent → own child via Guardian.userId; teacher → own class via classTeacherId). Pending pool is row-scoped for teachers.
- Restriction enforced for real: approval requires `student.edit`. Plain `TEACHER` only has `student.view`, so a teacher cannot silently change a photo/document — they raise a request a class teacher / department approves.
- Route fixed to `student.view` (read/submit) and `student.edit` (approve); typed error mapping added.
- Parent UI added: `ParentUploadCard` in `parent-portal-client.tsx` (encrypted FileUpload → POST `/api/students/approvals`, shows "Pending school approval").

**Proof:** `scripts/k10-approval-test.ts` — **6/6 green** (own-child allowed, non-child forbidden, teacher row-scope, photo lands on profile, document creates StudentDocument, re-review rejected). DB left clean.

## K.16 — KNEC Document Aggregation  ✅ REPAIRED & VERIFIED

**Problem found (over-claimed):** `KnecExportBatch` model existed but had **no service / route / UI / workflow** — no real aggregation or export.

**Fix (full-stack):**
- New service `knec-export.service.ts`: `createBatch` (target class + required labels), `aggregateBatch` (per-candidate completeness by matching `StudentDocument.label` to required labels, case-insensitive), `exportBatch` (refuses while incomplete unless forced; builds a `NEYO-KNEC-AGGREGATE-V1` manifest; stores it as an encrypted Storage-Vault artifact; sets `exportUrl`; flips status to EXPORTED).
- New storage helper `storeGeneratedArtifact` for server-generated, tenant-encrypted artifacts.
- New API `/api/exam-materials/knec-export` (GET list/aggregate, POST create/export).
- New UI `KnecAggregationCard` on the exam-timetable page (create batch, check completeness, export, download manifest).
- Deterministic, **no AI / Bundi dependency**.

**Proof:** `scripts/k16-knec-export-test.ts` — **10/10 green** (create, incomplete-before-uploads, export-blocked-while-incomplete, complete-after-uploads, EXPORTED flip, manifest stored + read back + every candidate's document URLs present). DB left clean.
