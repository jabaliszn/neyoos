# J.13 Repair — Parent Growth Dashboard
**Date:** 2026-06-29 · **Status:** ✅ Repaired to true full-stack, all 6 lines [x], re-verified by passing test.

## What was wrong (see J13-AUDIT)
The growth tab was IMPORTED but NEVER mounted (dead screen). Upcoming-assessments query used a non-existent status ("PUBLISHED") so it was always empty. No attendance/behavior/portfolio/feedback in the dashboard. Invalid Badge/Button props. Competencies shown without an approved-only filter. No school-enable gate for goal ack. Service also called tenantDb() without withTenant().

## What I built / changed (all 8 full-stack layers)

### 1. Database & migrations
- No new model needed. Used existing AttendanceRecord (status P|A|L|E), DisciplineIncident (severity/points/status), PortfolioItem (status APPROVED + visibleToParents), CompetencyEvidence (approved + visibleToParents + narrative), TalentRecord (score/notes/coach), StudentGoal (acknowledgedByParent/At), AssessmentPlan (status/dueDate/visibleToParents).
- Feature gate stored as a TenantModule row (moduleKey "parent_goal_ack"); default ON when no row.

### 2. Security & validation
- getParentGrowthDashboard and acknowledgeStudentGoal verify the caller is a guardian of the child (PARENT must have a StudentGuardian link); GrowthError("FORBIDDEN") otherwise. Everything runs inside withTenant() (was missing before). GrowthError mapped in respond.ts (404/403/403/422).
- Approved-only safety: competencies require approved + visibleToParents; portfolio requires status APPROVED + visibleToParents.

### 3. Backend services (src/lib/services/parent-growth.service.ts — rewritten)
- Attendance summary (last 60 days, present %), behavior summary (incidents + demerit points + severity split, APPROVED only), approved portfolio highlights, teacher feedback digest (approved competency narratives + coach/teacher talent notes + teacher goal descriptions), competency growth, talents, teacher goals, and FIXED upcoming assessments (status in ACTIVE/MODERATION + future dueDate, parent-visible, ordered by dueDate).
- "Growth not just grades" summary roll-up. isGoalAckEnabled() gate; acknowledgeStudentGoal refuses when disabled (GrowthError "DISABLED").

### 4. API endpoints
- GET /api/portal/parent/growth?studentId= and POST { action:"acknowledge_goal", goalId } (unchanged paths; service hardened; route returns the updated goal).

### 5/6. UI components & pages
- src/components/portal/parent-growth-tab.tsx rewritten: summary roll-up cards + Goals (with Acknowledge) + Attendance + Behavior + Talents + Competency growth + Portfolio highlights + Upcoming assessments + Teacher feedback digest. Removed invalid props (Badge now uses tone; no variant="outline").
- **CRITICAL FIX:** mounted <ParentGrowthTab studentId={id} /> inside ChildView in src/components/portal/parent-portal-client.tsx (under a "Growth dashboard" card). It was previously imported but never rendered.

### 7. Four UX states
Loading (spinner), error (retry button), empty (per-card empty text + whole-dashboard empty message), data (all cards).
