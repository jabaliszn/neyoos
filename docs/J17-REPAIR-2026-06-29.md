# J.17 Repair Report — Community Service Module
Date: 2026-06-29
Status: FULLY REPAIRED AND RE-VERIFIED

## What was not honest before
- J.17 had a real foundation, but it was not honestly full-stack.
- Evidence was only implied by `proofFileId`; it was not wired through validation, service, API and UI.
- Approval was only a raw status field, not a usable workflow.
- Community service was not proven to contribute to competencies.
- Community service was not present in learner journey aggregation.
- There was no report export or certificate export.
- The service tab was imported but not mounted in the student profile.
- The service layer used `tenantDb()` without `withTenant(...)`.
- Route permissions used invalid plural strings.
- UI still used invalid shared component props.

## What was repaired

### 1) Tenant-safe community service service
Rebuilt `src/lib/services/community-service.service.ts` to:
- wrap flows in `withTenant(user.tenantId, ...)`
- validate encrypted evidence file references
- support competency linking
- add approval decision flow
- create competency evidence on approved service records
- write audit logs for log / approve / reject / delete actions

### 2) Evidence support is now real
Updated validation and service support for:
- `proofFileId`
- encrypted Storage Vault check

This means evidence is now part of the J.17 full-stack flow.

### 3) Real approval workflow added
Added decision flow with:
- `communityServiceDecisionSchema`
- `decideServiceActivity(...)`
- API `POST` action `decision`

This now makes approval an actual workflow, not just a status value sitting in the DB.

### 4) Community service now contributes to competencies
When an activity is approved and linked to a competency:
- `CompetencyEvidence` is created with `sourceModule: "COMMUNITY"`
- it is approved and visible to parents

That makes the competency contribution real.

### 5) Community service now appears in learner journey
Updated:
- `src/lib/services/learner-journey.service.ts`

Now approved community-service records are added into the learner journey timeline, making this checklist claim true.

### 6) Report and certificate export added
Added:
- `src/lib/documents/community-service-report-pdf.tsx`

Extended route so it now exports:
- `format=pdf` → Community Service Report
- `format=certificate` → Community Service Certificate

Both are real PDF outputs.

### 7) UI repaired and mounted for real
Updated:
- `src/components/students/student-service-tab.tsx`
- `src/components/students/student-profile-client.tsx`

Now the student service tab is actually mounted and visible in the student profile.

UI improvements:
- report PDF button
- certificate PDF button
- evidence file ID field
- competency link field
- workflow status selection
- valid Badge/Button props

### 8) Permissions fixed
Route now uses correct singular permissions:
- `student.view`
- `student.edit`
