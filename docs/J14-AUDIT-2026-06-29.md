## Specific defects (real bugs, not just gaps)
1. **Discipline always empty.** `generateDigitalIdentitySnapshot` queries `disciplineIncident { status: "CLOSED" }`. DisciplineIncident status is `PENDING | APPROVED | REJECTED` — there is NO "CLOSED". So behavior never appears.
2. **Competency safety gap.** Snapshot filters competencies by `visibleToParents` but NOT `approved` — unapproved evidence could leak into a transfer.
3. **Weak access code.** `accessCode = Math.random().toString().slice(2,10)` — not cryptographically secure, and the file even imports an unused `cryptoRandomStringAsync`.
4. **No import/redeem workflow (line 4).** `/api/students/passport` GET only lists OUTGOING transfers. There is no endpoint for a receiving school to redeem an accessCode, no `importedAt`/`receivedBy`, no receiving-side audit log. Status never moves past PENDING in code.
5. **No real export artifact (line 2).** "Export Passport" just writes a `TransferPassportRequest` row with `payloadJson`. A non-NEYO school gets no downloadable PDF/JSON file.
6. **No actual identity VIEW (line 1).** The snapshot is only built as a transfer payload; the tab renders the transfer list, not a readable identity page. Claimed sub-items clubs / leadership / awards / certificates are NOT gathered (no such models; closest are LeavingCertificate, CommunityServiceActivity, ActivityCategory — none wired in).
7. **Invalid permission strings.** Route uses `students.view` / `students.manage`; correct typed permissions are singular `student.view` / `student.edit` etc. (latent bug class).
8. **Invalid UI props.** `<Badge variant="secondary"|"outline">` — Badge uses `tone`, not `variant` (same bug fixed in J.13).
9. **No withTenant + raw `new Error`** in the service (tenant-context risk + unmapped errors → generic 500s).

## Conclusion
Downgrade: line 5 stays `[x]`; lines 1,2,3,6 → `[~]`; line 4 → `[ ]`. Recommended full repair: secure crypto access code; fix discipline status + approved-only competencies; add a real export artifact (PDF/JSON download) for non-NEYO; build a receiving-school redeem/import endpoint (accessCode + expiry + status→COMPLETED + `importedAt`/receiver + audit log on BOTH generate and redeem/access); a readable Digital Identity VIEW that aggregates the holistic record incl. LeavingCertificate + CommunityServiceActivity + leadership/awards where modeled; fix permission strings; fix Badge props; wrap in withTenant; add a passing full-stack test.
