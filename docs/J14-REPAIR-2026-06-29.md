# J.14 Repair Report — Student Digital Identity & Transfer Passport
Date: 2026-06-29
Status: FULLY REPAIRED AND RE-VERIFIED

## What was broken before
- The checklist was marked complete, but the feature was not honestly full-stack.
- The service used `tenantDb()` without `withTenant(...)`.
- Permission strings were wrong (`students.view`, `students.manage`).
- Discipline snapshot queried `status: "CLOSED"`, which does not exist.
- Competency snapshot leaked parent-visible but unapproved evidence.
- The access code was weak (`Math.random().toString().slice(...)`).
- There was no real receiving-school redeem/import workflow.
- There was no real export artifact for non-NEYO schools.
- The UI had invalid `Badge`/`Button` props and no proper identity view.
- There was no import metadata on `TransferPassportRequest`.

## What we repaired
### 1) Database + migration
Added real import/audit workflow fields to `TransferPassportRequest`:
- `importedAt`
- `receivedById`
- `receivedByName`
- `lastAccessedAt`
- status index

Migration folder:
- `prisma/migrations/20260629111500_j14_transfer_passport_import_audit/`

### 2) Service layer repaired
Rebuilt `src/lib/services/digital-identity.service.ts` to be tenant-safe and domain-safe:
- wrapped flows in `withTenant(user.tenantId, ...)`
- added `DigitalIdentityError`
- fixed discipline filter to `APPROVED`
- fixed competency filter to `approved: true` AND `visibleToParents: true`
- generated stronger access codes with Node crypto
- added outgoing transfer listing
- added receiving-school redeem/import flow
- added audit logging for generate + redeem + import
- added export helper for printable passport PDF
- added readable portable learner identity snapshot
- included modeled holistic records like leaving certificate and approved community service where available

### 3) API repaired
`src/app/api/students/passport/route.ts`
- fixed permissions to `student.view` and `student.edit`
- supports generate and redeem flows
- returns proper mapped domain errors

Added PDF export route:
- `src/app/api/students/passport/export/route.ts`

### 4) UI repaired
Rebuilt `src/components/students/student-identity-tab.tsx`:
- real portable learner identity view
- outgoing passport list
- generate passport dialog
- receiving-school redeem dialog
- export PDF action
- valid `Badge tone=` and valid `Button variant=` usage
- data minimization explanation
- proper loading / empty / success states

### 5) Export artifact added
Created printable transfer passport PDF:
- `src/lib/documents/student-transfer-passport-pdf.tsx`
- includes learner profile, shared modules, academic highlights, competency/talent/portfolio summaries, attendance/discipline/wellbeing summary
- footer: `Powered by NEYO · neyo.co.ke`

### 6) Validation + error mapping
Updated:
- `src/lib/validations/digital-identity.ts`
- `src/lib/api/respond.ts`

Added redeem schema and error handling for expired / forbidden / invalid passport flows.
