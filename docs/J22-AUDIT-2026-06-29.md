---

## 1. What J.22 promises (plain English)

Make sure all student portfolio, learner-journey, and transfer data respects: who is
allowed to see what (per role), parent/student consent, how long sensitive data is kept,
that sensitive medical/discipline data is not transferred unless explicitly approved,
alignment with Kenya's Data Protection Act / ODPC, and a full audit trail for exports,
transfers and report generation.

## 4. Specific defects found

1. **Dead retention engine:** `enforceDataRetentionPolicies()` is never invoked anywhere — not in `src/lib/jobs/registry.ts`, not in any route. It is unreachable.
2. **No export audit:** `exportTransferPassportPdf` / the export route drop no `AuditLog` entry, contradicting line 6.
3. **Export permission too loose:** PDF export is gated by `student.view` (TEACHER + PARENT have it). The old audit doc explicitly said only Admin/Principal should export.
4. **No portfolio-sharing consent:** line 2's portfolio half has no consent record/flow.
5. **No enforced ODPC lawful-basis logic:** only comments; nothing the system enforces beyond minimization.
6. **Dishonest test:** `scripts/j22-compliance-test.ts` re-implements retention inline instead of calling the service — it cannot fail even if the real service is broken.
7. **Corrupted checklist headers:** J.22 (and J.23) headers are duplicated/garbled.

---

## 5. Recommendation

Downgrade J.22 to **PARTIAL** and offer the founder:

- **Option 1 — Fix J.22 fully:**
  - clean the corrupted J.22 header
  - wire `enforceDataRetentionPolicies()` into the jobs registry (real cron) + give it `withTenant`-safe behaviour and an admin-triggerable API
  - add audit logging to passport PDF **export** + report generation
  - tighten the export route to Admin/Principal-level permission
  - add a portfolio-sharing **consent** record/flow (or honestly scope line 2 to transfer-only)
  - add an enforced ODPC data-minimization/lawful-basis assertion the code actually checks
  - gate the J.22 surface with the Part-J feature toggle (`assertJFeatureEnabled("J.22")`) and add `J.22` to `src/lib/core/j-features.ts`
  - replace the dishonest test with a **real** full-stack test that calls the actual services
  - then repair doc + labelled mockup + context anchor update
- **Option 2 — Accept partial:** keep it honestly `[x]`/`[~]` and move on.

No repair has been made yet — this document is the audit only.
