## 4. Specific defects found

1. **Enforcement gap (the big one):** only **1 of 6** revenue features (advanced analytics) actually checks entitlement. The other five are free to everyone вЂ” so the "packaging" is cosmetic pricing, not revenue protection.
2. **Add-on entitlement bug:** `requirePremiumFeature` only checks `plan.includedModules`. A school that **buys** the `skills_passport` (or other) add-on Г  la carte вЂ” stored on `subscription.addOns` вЂ” would still be **denied**, because the gate never reads `addOns`. So even where a gate exists, paying Г  la carte wouldn't unlock the feature.
3. **`TierGatingError` not centralized:** it is only caught inside the analytics route. If reused elsewhere without a local try/catch it would surface as a 500 instead of a clean 402.
4. **Weak/dishonest-ish test:** `scripts/j23-tier-gating-test.ts` covers a single feature, mutates seed state (leaves the tenant on `elite`), ignores the add-on path, and swallows failures (`catch(console.error)`).
5. **Corrupted checklist header.**
