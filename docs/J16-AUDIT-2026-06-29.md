## Honest line-by-line verdict

### 1) Competency gap analytics by class/grade/subject/teacher
Current implementation in `advanced-analytics.service.ts` only does this:
- fetches competency evidence with level 1 or 2
- counts by competency ID/name
- returns top 5 weak competencies overall

What is missing from the checklist claim:
- **by class** ❌
- **by grade** ❌
- **by subject** ❌
- **by teacher** ❌

This is a real weak-competency list, but **not** the full dimensional analytics claimed.

Verdict: **`[~]` partial**.

### 2) Per-subject/per-teacher academic performance analytics foundation
This one is real.

`exam-analytics.service.ts` does:
- subject means
- teacher-linked performance via class-subject teacher mapping
- learner progress over time
- term trend summaries

API and mounted UI also exist.

Verdict: **`[x]` real**.

### 3) Assessment balance analytics
This exists in a basic real form.

`advanced-analytics.service.ts` counts:
- `exam.count()`
- `assessmentPlan.count()`
- approved `portfolioItem.count()`

That is enough to support a real “exam-heavy vs flexible/project/portfolio” signal.

Caveat:
- “practicals” are not separately analyzed; they are folded into assessment plans indirectly.

Still, as a foundation, this is real.

Verdict: **`[x]` acceptable real foundation**.

### 4) Attendance-to-performance correlation
This exists in real code.

The service:
- pulls student absences
- computes average exam percentages
- groups into absence brackets
- returns trend data for leadership view

Verdict: **`[x]` real**.

### 5) Talent participation and wellbeing indicators
Talent participation is real.

The service:
- groups talent records by talent area
- returns participation counts

But “wellbeing indicators” is overstated.
Current implementation does **not** really compute broader wellbeing indicators beyond intervention hints and attendance risk. There is no dedicated wellbeing index, behavior-wellbeing blend, or counseling/health style indicator in this J.16 service.

Verdict: **`[~]` partial**.

### 6) Pathway readiness analytics
Not honestly part of this J.16 analytics surface yet.

Yes, pathway readiness exists elsewhere in the repo under J.10/J.18-related pathway services and pathway report export, but the J.16 advanced analytics service does **not** aggregate pathway readiness into this dashboard output.

So the checklist line is over-claiming if it means this section is implemented here end-to-end.

Verdict: **`[~]` partial**.

### 7) Principal dashboard cards for weak competencies and intervention needs
This is real in a basic but honest way.

The advanced analytics service creates intervention cards for:
- attendance risk
- assessment imbalance
- systemic competency gap

The UI renders these as leadership intervention alerts.

Verdict: **`[x]` real**.

## Real bugs and hardening problems found

### 1) Tenant safety bug
`src/lib/services/advanced-analytics.service.ts` uses `tenantDb()` directly but is **not wrapped in `withTenant(user.tenantId, ...)`**.

That is the same recurring service bug we have found in multiple Part J sections.

### 2) Route bug: missing `fail` import
`src/app/api/analytics/advanced/route.ts` calls:
- `fail("PAYMENT_REQUIRED", ...)`

but only imports:
- `ok, handleError`

So this route has a real code bug.

### 3) Permission/mount mismatch risk
The advanced analytics UI is mounted on the Exams page, which is guarded by `exam.view`, but the API route itself requires `reports.view`.

This means some users may successfully load the page but fail the analytics call depending on permission shape.

That is not necessarily fatal, but it is a product inconsistency and weakens the claim of a polished full-stack section.

### 4) UI prop bug
`advanced-analytics-client.tsx` uses:
- `<Badge variant="destructive">`

That does not match the known shared Badge contract for this repo.

### 5) Page mount gap
`AdvancedAnalyticsClient` is imported on Exams page, but currently not rendered in the returned JSX.
So the J.16 advanced analytics surface is effectively **not visible** there right now.

That is a major honesty problem for a section marked complete.
