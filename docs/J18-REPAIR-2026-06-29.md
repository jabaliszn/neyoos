# J.18 Repair Report — Career Discovery & Pathway Guidance
Date: 2026-06-29
Status: FULLY REPAIRED AND RE-VERIFIED

## What was not honest before
- J.18 had a real record-logging foundation, but it was not honestly full-stack.
- There was no true rule-based mapping engine combining interests, competencies, performance and talents into career areas.
- The career tab existed in code but was not mounted in the student profile.
- The service used `tenantDb()` without `withTenant(...)`.
- The route used invalid plural permission strings.
- The UI still used invalid shared-component props.
- The parent/student conversation view existed only as raw records, not a visible dedicated section.

## What was repaired

### 1) Tenant-safe career discovery service
Rebuilt `src/lib/services/career-discovery.service.ts` to:
- wrap flows in `withTenant(user.tenantId, ...)`
- keep record logging and deletion
- add real `getCareerDiscoveryProfile(...)`
- expose a rule-based recommendation mode: `RULE_BASED_NO_BUNDI`

### 2) Real rule-based mapping engine added
Built a real rules engine that now combines:
- logged interests and recommendation notes
- approved competency evidence
- exam-based subject averages
- talent records
- pathway readiness support signals

It produces ranked career matches with:
- area
- score
- reasons
- confidence

This makes the “map interests + competencies + performance + talents to career areas” claim true.

### 3) Route upgraded
Updated `src/app/api/students/careers/route.ts` to:
- use correct singular permission string `student.view`
- expose `view=profile` mode
- return the full rule-based profile payload

### 4) UI repaired and mounted for real
Updated:
- `src/components/students/student-career-tab.tsx`
- `src/components/students/student-profile-client.tsx`

Now the career tab is actually visible in the student profile and includes:
- rule-based career matches
- confidence badges
- reasons for each match
- parent/student conversation view
- signals used for guidance (academic, competency, talent)
- timeline of logged career records

### 5) Bundi rule respected
The repaired engine is explicitly rule-based and does not depend on Bundi/AI.
The profile returns:
- `mode: RULE_BASED_NO_BUNDI`

So the checklist line is now honest.
