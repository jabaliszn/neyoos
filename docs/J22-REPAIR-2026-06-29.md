## 2. What we changed (plain English)

| Fix | File | What it now does |
|---|---|---|
| **Wired the real retention engine into the daily cron** | `src/lib/jobs/registry.ts` | The `data-retention` job (runs daily 04:00 EAT) now actually calls `enforceDataRetentionPolicies()`. Expired transfer-passport payloads are wiped (data minimisation) and stale unapproved portfolio evidence is purged — automatically. No more dead code. |
| **Enforced ODPC lawful-basis guard** | `src/lib/services/retention.service.ts` (`assertLawfulTransferBasis`) | A transfer **cannot be created** without a named consenting guardian, and only valid, de-duplicated data modules may be carried. | **Recorded the consent timestamp** | `src/lib/services/digital-identity.service.ts` | Every transfer now stores `consentDate` alongside `consentBy` (proof of when consent was given). |
| **Audit log on PDF export** | `src/lib/services/digital-identity.service.ts` | Exporting a transfer-passport PDF now writes a `compliance.transfer_passport_exported` audit entry (who, role, modules, format). |
| **Tightened export permission** | `src/app/api/students/passport/export/route.ts` | PDF export changed from `student.view` (TEACHER + PARENT) to `student.edit` (Admin/Principal only). |

| **Wired the new error type** | `src/lib/api/respond.ts` | `ComplianceError` → 403 (FORBIDDEN) / 400 (INVALID). |
| **Replaced the dishonest test** | deleted `scripts/j22-compliance-test.ts`; added `scripts/j22-compliance-fullstack-test.ts` | The new test calls the **real** services and the **real** cron job handler — it cannot pass if the real code is broken. |
