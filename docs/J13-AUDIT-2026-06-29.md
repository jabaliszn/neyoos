# J.13 Audit — Parent Growth Dashboard
**Date:** 2026-06-29 · **Auditor:** Agent (full-stack honesty audit) · **Verdict:** ❌ Dishonestly marked COMPLETED → ✅ REPAIRED 2026-06-29 (see J13-REPAIR-2026-06-29.md)

## Headline
A `ParentGrowthTab` component, a `parent-growth.service.ts`, and a `/api/portal/parent/growth` route all exist — but the tab is **imported and NEVER rendered** in `parent-portal-client.tsx`. So the entire growth dashboard is **unreachable by parents**. On top of that, several claimed data layers are missing, and there are real bugs.

## Evidence / specific defects
1. **Not mounted.** `parent-portal-client.tsx` line 30 imports `ParentGrowthTab` but there is no `<ParentGrowthTab .../>` anywhere in the file (grep: only the import). The `ChildView` renders Fees/Results/Attendance/Timetable/Homework/Class-notes/Talk-to-school/Pickup/competency-summary — never the growth tab. Dead code.
2. **Upcoming assessments always empty (real bug).** `parent-growth.service.ts` queries `assessmentPlan.findMany({ where: { status: "PUBLISHED", ... } })`. AssessmentPlan status enum is `DRAFT | ACTIVE | MODERATION | RELEASED | ARCHIVED` — there is **no "PUBLISHED"** value, so the result is always `[]`. Also it filters `classId` + `visibleToParents` but never filters on a future `dueDate`, so it is not really a "calendar".
3. **No attendance/behavior in the growth dashboard.** Service returns only goals/talents/competencies/upcoming. (Attendance IS shown elsewhere in ChildView, but behavior/discipline is not surfaced to parents at all, and neither is part of the "growth dashboard" the checklist describes.)
4. **No portfolio highlights.** PortfolioItem (status APPROVED) is never queried by the growth service; no UI card.
5. **No teacher feedback digest.** No aggregation of teacher remarks/feedback anywhere in the growth service or tab.
6. **Invalid UI props.** The tab uses `<Badge variant="outline"|"secondary">` and `<Button variant="outline">`. Badge has no `variant` prop (it uses `tone`); Button variants are `primary|secondary|ghost|danger` (no `outline`). These render as default/ignored and are a code-quality bug.
7. **Approved-only safety gap.** Competency query filters `visibleToParents: true` but not `approved: true`; the J.12 rule and "view safely" intent imply approved-only.
8. **No "school enables it" gate** for goal acknowledgement (line 6 says "where school enables it") — there is no tenant feature flag check.

## Conclusion
Downgrade: lines 1–5 → `[ ]`, line 6 → `[~]`. Recommended repair: build the missing data (attendance summary, behavior summary, approved portfolio highlights, teacher feedback digest), fix the upcoming-assessments bug (use real status + future dueDate), add a "growth not just grades" summary roll-up, fix invalid Badge/Button props, enforce approved-only, add a school-enable flag for goals, and — critically — **mount the tab** in the parent portal. Then a passing full-stack test + `npm run test:roles`.
