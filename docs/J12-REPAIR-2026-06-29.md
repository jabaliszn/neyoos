# J.12 Repair — Teacher Planning Linked to Curriculum Objectives
**Date:** 2026-06-29 · **Status:** ✅ Repaired to true full-stack, all 6 lines [x], re-verified by passing test.

## What was wrong (see J12-AUDIT)
Backend/schema mostly real, but: no UI to pick curriculum links, no resource-attach UI, observations-from-lesson-plan never existed, analytics had no "assessed" dimension and the analytics endpoint was never mounted in any screen.

## What I built / changed (all 8 full-stack layers)

### 1. Database & migrations
- New model LessonObservation (prisma/schema.prisma): lessonPlanId, optional studentId (null = whole-class), optional strandId/competencyId, optional level (1-4 CBC), note, teacherId/teacherName, date. Back-relation observations on LessonPlan; relation on Tenant.
- Registered lessonObservation + lessonResource in src/lib/core/tenant-tables.ts (tenantDb auto-stamps + isolates tenant).
- Migration prisma/migrations/20260629090000_j12_lesson_observation/migration.sql (resolved as applied; dev DB synced via db push).

### 2. Security & validation
- lessonObservationSchema in src/lib/validations/academics.ts (cuid ids, level 1-4, note 2-1000 chars).
- Service guards: teachers can only observe/attach to their own plans; named learner must exist and be in the lesson's class (else INVALID). Extended AcademicsError with "INVALID".

### 3. Backend services (src/lib/services/academics.service.ts)
- recordLessonObservation, listLessonObservations, addLessonResources (all own-scoped + audit-logged).
- getLessonPlanningAnalytics rewritten: teacher-scoped; returns planned / taught (taughtPct) / assessed (assessedPct) plus strand coverage %, competencies taught, plans linked to assessment, objectives assessed. "Assessed" = plan linked to an AssessmentPlan that has recorded AssessmentRecord rows (fixed a real bug: now groups by the correct planId field).

### 4. API endpoints
- GET/POST /api/academics/lesson-plans/observations
- POST /api/academics/lesson-plans/resources
- (existing) GET /api/academics/lesson-plans/analytics — now reachable from UI.

### 5/6. UI components & pages (src/components/academics/academics-client.tsx -> LessonsTab)
- PlanDialog: curriculum link pickers (strand /api/cbc/strands, competency /api/competencies, assessment plan /api/assessments) + resource-attach field.
- Each plan row: curriculum-link badges + resource count + Observe / Resources / Coverage actions.
- ObservationDialog, ResourceDialog, CoverageDialog, Stat panel. Tab mounted under Academics -> "Lesson plans".

### 7. Four UX states
Loading (Skeletons), error (LoadError + retry), empty (EmptyState), data (table + dialogs).

### 8. Kenyan seed data
prisma/seed.ts: Chebet's "Quadratic equations — completing the square" plan links to Mathematics strand "Algebraic Expressions", has a worked-video resource, and a CBC level-3 observation.

## Parent-facing guarantee (line 6)
Lesson plans and raw teacher observations are NOT exposed by any parent portal API/component (grep of src/app/api/portal/ and src/components/portal/ for lesson/observation -> none). Parents only see approved/high-level progress (J.13 + approved competency/portfolio). Rule holds.
