# J.21 — NEYO Ops Curriculum Template Library — Honest Audit
**Date:** 2026-06-29
**Auditor action:** Audit only (no repair yet). Source of truth = real repo state, not the checkmarks.
**Headline verdict:** ❌ **Marked COMPLETED too generously.** Should be downgraded to **PARTIAL**.

---

## 1. What J.21 promises (plain English)

NEYO (the company) maintains official curriculum templates (e.g. CBC Kenya, 8-4-4 legacy, private-school templates). Schools can browse this library, copy a template into their own school, and customise it locally. When NEYO updates a template, schools should be told and can choose to adopt the update.

## 4. Specific defects found

1. **Tenant-isolation bug:** `adoptCurriculumTemplate` uses `tenantDb()` outside any `withTenant(user.tenantId, ...)` block — same class of bug fixed in J.17–J.20.
2. **Generic errors:** the service throws raw `new Error("Unauthorized")` instead of a typed error class wired into `handleError` — so the API returns 500 with a generic message instead of a proper 403/404.
3. **No publish/create audit:** `upsertGlobalTemplate` writes nothing to `AuditLog`, contradicting line 5.
4. **No NEYO Ops UI:** nothing in `src/components` or `src/app` renders or calls `/api/ops/curriculum-templates`. NEYO staff literally cannot use this feature through the product.
5. **No "announce updates" path:** line 4's "announced" half is entirely unbuilt.
6. **Dishonest test + dirty data:** `scripts/j21-global-curriculum-test.ts` is a direct-DB poke and has left duplicate `PUBLISHED` rows in the dev DB.
7. **Double-wrapped API responses:** ops route returns `ok({ data: templates })` (the same wrapper bug fixed elsewhere), so any future UI would read `json.data.data`.
8. **Checklist header note:** J.22's header on the checklist is corrupted/duplicated (`## J.22 — Compliance, Consent ## J.22 — ... Data Safety (COMPLETED)`) — flagged for the J.22 audit later.

---

## 5. Recommendation

Downgrade J.21 to **PARTIAL** and offer the founder the standard choice:

- **Option 1 — Fix J.21 fully:** add `withTenant` + typed errors + publish/create audit logging, build a real NEYO Ops template-management UI with 4 UX states, add a lightweight "update available / announce" signal so schools can adopt updates intentionally, fix the response wrapper, replace the dishonest test with a real full-stack test, clean the duplicate rows, then document + mockup + context anchor.
- **Option 2 — Accept partial:** keep it honestly marked `[~]`/`[ ]` and move on.

No repair has been made yet — this document is the audit only.
