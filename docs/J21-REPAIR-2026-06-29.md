# J.21 — NEYO Ops Curriculum Template Library — Repair Report
**Date:** 2026-06-29
**Action:** Option 1 — Fix J.21 fully (after honest audit + downgrade) **+** founder add-on: Part-J feature ON/OFF toggles in NEYO Ops (default ON).
**Result:** ✅ Repaired to true full-stack and re-verified.

---

## 1. What J.21 promises (plain English)

NEYO (the company) keeps official curriculum templates (CBC Kenya, 8-4-4 legacy, private-school). Schools browse this library, copy a template into their own school as a draft, and customise it. When NEYO updates a template, schools are told and can adopt the update on purpose.

---

## 2. Honest state BEFORE repair (from the audit)

- Marked COMPLETED too generously → downgraded to PARTIAL.
- `adoptCurriculumTemplate` used `tenantDb()` **without** `withTenant(...)` (tenant-isolation bug).
- Service threw raw `new Error("Unauthorized")` instead of typed errors → wrong HTTP codes.
- **No publish/create audit log** (only adoption was logged).
- **No NEYO Ops UI at all** — staff could not use the feature in-product.
- **No "announce updates" mechanism** — line 4 was entirely unbuilt.
- The old test `scripts/j21-global-curriculum-test.ts` was **dishonest** (direct DB insert) and had leaked **2 duplicate published rows** into the dev DB.
- API responses were double-wrapped (`ok({ data })`).

---

## 3. What was fixed (full-stack, all 8 layers)

| Layer | Change |
|---|---|
| 1. DB & migrations | `GlobalCurriculumTemplate`: added `publishedAt`, `changeNote`, `announcedAt`. `Curriculum`: added `adoptedTemplateId`, `adoptedTemplateVersion` so a school remembers which template + version it adopted. Pushed to the SQLite dev DB. |
| 2. Security & validation | `global-curriculum.ts` adds `changeNote`. Service uses typed `GlobalTemplateError` (FORBIDDEN/NOT_FOUND/INVALID/CONFLICT) wired into `handleError` (proper 403/404/409/422). Ops routes guard `requireRole("SUPER_ADMIN")`; school routes guard `academics.view` / `academics.manage`. |
| 3. Backend services | `global-curriculum.service.ts` rewritten: `withTenant(...)` for adoption; audit logs for **create / publish / update / announce / delete / adopt**; `publishedAt` stamped on first publish; `announceTemplateUpdate`; `getTemplateUpdatesForSchool` (detects newer published version vs adopted version). |
| 4. API endpoints | `/api/ops/curriculum-templates` (GET/POST/DELETE + `?action=announce`), response shape fixed (`ok(data)`). `/api/curriculum/library` (GET list / `?view=updates` / POST adopt), gated by the J.21 feature toggle. New `/api/ops/j-features` (GET list / POST toggle). |
| 5. UI components | New NEYO Ops tabs in `founder-ops-client.tsx`: **Curriculum Library** (create / publish / announce / delete with 4 UX states) and **Feature Toggles** (Part-J ON/OFF with 4 UX states). School-side **update-available banner** added to `curriculum-version-manager.tsx`. |
| 6. Mounted surface | Both tabs mounted in the SUPER_ADMIN Founder Operations console (`/founder`). |
| 7. Four UX states | Loading (skeletons), error (with retry), empty, and data — in both new Ops tabs. |
| 8. Kenyan seed data | Test exercises real `karibu-high` principal + `support@neyo.co.ke` SUPER_ADMIN; uses CBC Kenya / KICD Junior School context. Dirty duplicate rows from the old test were cleaned. |

### Founder add-on — Part-J feature toggles
- New registry `src/lib/core/j-features.ts` lists J.14–J.21.
- Reuses the existing `PlatformFlag` table with `jfeature:<id>` keys. **Default ON** (a feature is OFF only if an explicit paused flag exists).
- Service: `listJFeatureFlags`, `isJFeaturePaused`, `assertJFeatureEnabled`; `setFlag` now accepts `jfeature:` keys and audit-logs `platform.jfeature_paused` / `_released`.
- The J.21 school library is gated by `assertJFeatureEnabled("J.21")` as the first wired example.
